# oc-spec-rules

Actionable rules for the SDD spec phase. Kept compact for executor context.

## Rules

- Capture WHAT the change must do, never HOW — the design phase decides HOW.
- Every requirement must be testable and map 1:1 to a verification scenario.
- Make acceptance criteria numbered and unambiguous.
- Record what is explicitly out of scope.
- Do NOT start implementation in this phase.
