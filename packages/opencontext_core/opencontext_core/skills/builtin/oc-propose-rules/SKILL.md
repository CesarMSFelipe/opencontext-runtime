# oc-propose-rules

Actionable rules for the SDD propose phase. Kept compact for executor context.

## Rules

- Turn explore findings into a concrete, bounded proposal — scope and sequence only.
- State explicit non-goals to prevent scope creep in later phases.
- Do NOT implement or edit code in this phase; this is planning input.
- Ground every claim in real symbols already discovered during explore.
- Prefer the smallest change that satisfies the task.
