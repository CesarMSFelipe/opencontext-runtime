# oc-tasks-rules

Actionable rules for the SDD tasks phase. Kept compact for executor context.

## Rules

- Slice the design into ordered, independently verifiable work items.
- Each task must name the files it touches and its acceptance check.
- Order tasks so each leaves the suite green; no task depends on a later one.
- Keep tasks small — one logical work unit each.
- Do NOT implement in this phase; produce the plan only.
