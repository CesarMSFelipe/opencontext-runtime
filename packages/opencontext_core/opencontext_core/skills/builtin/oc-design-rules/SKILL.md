# oc-design-rules

Actionable rules for the SDD design phase. Kept compact for executor context.

## Rules

- Turn the spec into a concrete technical design the apply phase can follow without guessing.
- Reuse existing symbols before adding new ones; justify every new component.
- Make trade-offs explicit and prefer the simplest design that meets the spec.
- Specify files to create or modify, data flow, and the testing strategy.
- No code edits in this phase.
