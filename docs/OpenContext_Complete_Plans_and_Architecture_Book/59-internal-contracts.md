# 59 — Internal Contracts

Authority: architecture review §3 (imprescindible #3) + §4 (internal versioning), §6 (Brain
restrictions), §7 (event hierarchy), §8 (global IDs). Public contracts (doc 17) are for plugins
and external agents; **internal contracts** keep the runtime maintainable across tens of thousands
of lines. Internal contracts are Python `Protocol`s living in L0, versioned, and depended on by
upper layers (never the reverse).

## Internal API surfaces (Protocols)

| Internal API | Owning PR | Key methods (indicative) |
|--------------|-----------|--------------------------|
| Runtime API | PR-001 | `start_session, run, next, observe, apply, inspect, resume, archive` |
| Context API | PR-010 | `build_envelope(task, budget) -> ContextEnvelope`, `rank(items)`, `compress(ctx)` |
| Scheduler API | PR-000.1 | `decide_next(state) -> NextNodeDecision`, `simulate(plan) -> SimulationReport` |
| Persona API | PR-006 | `resolve(role, profile) -> PersonaDefinition`, `handoff(...)` |
| Skill API | PR-006 | `resolve(id) -> SkillDefinition`, `bundle(persona)`, `run(skill, inputs)` |
| Harness API | PR-006/007 | `run(harness, ctx) -> HarnessResult`, `gates(harness)` |
| Cache API | PR-000.3 | `get(key) -> Entry|None`, `put(key, value, provenance)`, `invalidate(rule)` |
| Decision API | PR-000.4/000.1 | `record(decision) -> DecisionLogEntry`, `log_for_run(run_id)` |

Rule: upper layers depend on the **Protocol**, not the concrete class — enables swapping
implementations and the compatibility-layer adapters (`pr-000-0`) without touching callers.

## Internal contract versioning (review §4)

Every internal contract carries an internal version, bumped on breaking change; the
compatibility layer maps across versions.

```
Runtime Contract v1
Workflow Contract v1
Context Contract v1
Provider Contract v1
Decision Contract v1
Cache Contract v1
Studio Contract v1
Plugin Contract v1   (this one is ALSO public — doc 17)
```

Stored as a constant per contract module (e.g. `RUNTIME_CONTRACT_VERSION = 1`) and asserted by a
guard test so an accidental breaking change is caught.

## Global IDs (review §8)

Everything is addressable. One ID scheme, prefixed, sortable (ULID-style time-ordered where the
entity is event-like). No `Date.now()`/random in scripts — generated by a single `ids.py` factory.

```
SessionID    sess_<ulid>
RunID        run_<ulid>
WorkflowID   wf_<slug>           NodeID      node_<slug>
DecisionID   dec_<ulid>          ReceiptID   rcpt_<ulid>
ArtifactID   art_<ulid>          MemoryID    mem_<ulid>
KGNodeID     kg_<hash>           CacheEntryID cache_<hash>
ProviderCallID pcall_<ulid>
```

Hash-based for content-addressed entities (KG nodes, cache entries — deterministic, dedup-safe);
ULID for event-ordered entities (sessions, runs, decisions, receipts, provider calls). Owned by
PR-001 `runtime/ids.py`; reused everywhere.

## Event hierarchy (review §7)

Every `RuntimeEvent.type` belongs to exactly one family. Studio filters by family.

```
lifecycle | workflow | context | memory | kg | runtime | policy | provider | studio | plugin | benchmark
```

`RuntimeEvent.family` is a required enum field (PR-001/PR-002); Studio (PR-014) renders one lane
per family. Replaces the flat event-name soup so the timeline is trivial to build.

## Runtime Brain restrictions (review §6) — Decision API invariant

The Runtime Brain (PR-000.1) **only decides**. It is structurally forbidden from side effects:

- MUST NOT modify files.
- MUST NOT invoke tools / run commands.
- MUST NOT write memory.
- MUST NOT skip or weaken harnesses.
- MUST NOT govern transitions — it *recommends*; the deterministic State Machine (PR-001) governs.

Enforced by capability: the Brain is constructed with **no** mutation/tool/memory ports — only
read ports (Intelligence, KG, history) and a `record(decision)` sink. A test asserts the Brain
class exposes no write-capable dependency. This keeps the Brain small (review §6).
