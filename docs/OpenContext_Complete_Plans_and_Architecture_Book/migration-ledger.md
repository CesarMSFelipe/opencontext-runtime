# Migration Ledger — Legacy ↔ Runtime vNext (PR-000.0)

Machine-readable mirror of the seeded ledger in
`packages/opencontext_core/opencontext_core/compat/migration.py`. The Python
`MIGRATION_LEDGER` is the source of truth; this table is generated by
`MigrationLedger.to_markdown()` so it cannot drift. The (deferred, PR-017)
release-gate lint reads this same ledger.

States: `legacy` (only legacy path) · `adapted` (vNext adapter exists behind a
flag; legacy still default) · `migrated` (vNext-only reach, legacy shimmed, parity
green, no direct importers) · `removed` (legacy deleted). A module's state reflects
whether its superseding PR has shipped (program build status:
PR-001/000/003/000.1/002/005/000.2/006/004/007 done; 008–017 + 000.3/000.4 pending).

## Two-spine convergence (CL-008)

Decision: **`HarnessRunner→RuntimeApi`** is the adopted vNext spine. `OcNewConductor`
is the narrower, later spine — marked `adapted` and scheduled for removal at
**milestone-C** once its resume carry-over behaviour reaches parity on the
HarnessRunner spine. RuntimeApi already brackets `HarnessRunner.run()`
(`runtime/api.py`), so it is the proven spine; recorded as `TWO_SPINE_CONVERGENCE`.

## "migrated" criteria (CL-006)

A module is `migrated` only when all four conditions hold (evaluable via
`compat.is_migrated(module) -> (bool, reasons[])`):

- (a) reachable only through the vNext contract — ledger fact `vnext_only`
- (b) legacy path removed/shimmed — ledger fact `legacy_shimmed`
- (c) parity tests exist — ledger field `parity_test` (existence-verified when a repo root is passed)
- (d) no module imports the legacy symbol directly — best-effort `ast` scan over the package
  (the rule the PR-017 release-gate lint will later enforce repo-wide; CL-013 deferred)

> folds the former `compat.py` version shim — `UTC`, `StrEnum`). The design's
> `runtime/migration.py` path was consolidated into `compat/` to keep the entire
> migration-governance surface behind one boundary.
