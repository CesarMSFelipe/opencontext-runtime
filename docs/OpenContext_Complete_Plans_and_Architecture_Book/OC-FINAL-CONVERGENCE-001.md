# OpenContext Final Convergence Instructions for the Development Agent
## Version 1.0
### Document ID
OC-FINAL-CONVERGENCE-001

---

# 1. Purpose

This document tells the development agent exactly what must be reviewed, reconciled and added before implementing OpenContext from the current branch.

It is based on the original architecture plans, the complementary plans, the complete architecture book, the PR sequencing plan, the PR-001/002/003 implementation specs, and the SDD/PR specs generated by the orchestration agent.

The goal: **nothing from the original OpenContext product vision is left behind.** After convergence, OpenContext must be ready to move from architecture/planning into real developer exploitation: incremental PRs, working runtime, stable SDD, OC Flow, benchmarks, first-run usability.

---

# 2. Non-Negotiable Instruction

Do **not** start broad implementation before doing this convergence pass. The current PR specs are valuable but incomplete vs the full target architecture.

Reconcile them against the architecture book and add the missing cross-cutting PRs/requirements. Do not rewrite the existing PR plan from scratch. Instead: (1) preserve the current PR sequence; (2) add missing foundational PRs; (3) patch the existing PR specs with missing requirements; (4) add acceptance criteria and benchmarks for every gap; (5) ensure the final roadmap leads to an exploitable developer product.

# 4. Main Problem Found

Current orchestration is `PRs -> SDD per PR -> implementation`. The target requires:

```
Intent -> Meta Planning -> Runtime Brain -> Scheduler -> Workflow Selection ->
Context/KG/Memory -> Harnessed Execution -> Runtime Intelligence ->
Artifacts/Receipts -> Benchmarks -> Learning Loop
```

The plan is strong on SDD docs, weaker on the layer that makes OpenContext a Gentle-AI-like engineering runtime. Close that gap.

## PR-000 — Meta Planning & Intent Engine
Modules: `planning/{intent,program,decomposition,pr_plan,risk,estimates}.py`.
Contracts: `IntentRecord, ProgramPlan, ImplementationSlice, PrPlan, RiskAssessment, ConvergenceMap`.
Responsibilities: parse intent; map intent→architecture docs; generate PR slices; deps; risk; cost; recommend workflow per slice; ensure every architecture requirement is assigned to a PR.
Acceptance: PR coverage map producible; every requirement assigned/deferred-to-1.x/rejected-with-reason; no orphaned requirement; ProgramPlan persisted as artifact + receipts.
Refs: 01, 16, 43, 49, 45.

## PR-000.1 — Runtime Brain & Scheduler
Modules: `runtime/{brain,scheduler,decisions,execution_strategy}.py`.
Contracts: `RuntimeDecision, DecisionLog, SchedulingDecision, ExecutionStrategy, NextNodeDecision`.
Responsibilities: decide next node/persona/skill-bundle/harnesses/context-strategy/provider/profile/retry-policy; adaptive-but-not-opaque; emit decision receipts; expose to Studio/CLI.
Acceptance: every selection = a `RuntimeDecision`; persisted in Decision Log; Brain recommends, State Machine governs; decisions inspectable from CLI/MCP; no hidden agent-only orchestration.
Refs: 02, 11, 24, 45, 50.

## PR-000.2 — Capability Graph & Execution Profiles
Modules: `capabilities/{graph,detector,registry,constraints}.py`, `profiles/{definition,resolver,strategy}.py`.
Contracts: `CapabilityNode, CapabilityGraph, CapabilityConstraint, ExecutionProfile, ExecutionProfileStrategy`.
Profiles: balanced, low-cost, enterprise, research, performance. Strategies: fast, cheap, careful, enterprise, research, local_first.
Responsibilities: detect local tooling; capability dependencies; adapt workflows to capabilities; configure budgets/retries/harness-strictness/provider-routing; first-run works without manual setup.
Acceptance: `opencontext doctor` builds a Capability Graph; workflow selection uses capability availability; OC Flow degrades gracefully when tests/lint missing; profiles influence context budget/diagnosis attempts/harness modes; missing capabilities → actionable UX.
Refs: 13, 21, 23, 36.

## PR-000.3 — Semantic Cache & Runtime Optimizer
Modules: `cache/{semantic_cache,tool_cache,ast_cache,provider_cache,invalidation}.py`, `optimization/{optimizer,recommendations}.py`.
Contracts: `SemanticCacheEntry, ToolCacheEntry, AstCacheEntry, ProviderCacheEntry, CacheInvalidationRule, RuntimeOptimizationRecommendation`.
Cache types: Semantic, Prompt/Context, Tool Output, AST, Provider Response, KG Query, Memory Retrieval.
Responsibilities: avoid repeated KG/memory retrieval, AST parsing, tool calls, prompt construction; track hit/miss; feed Runtime Intelligence.
Acceptance: cache policy-aware; invalidates on file changes; entries have provenance; improves token/tool metrics in benchmark; optimizer recommends cache/context/profile improvements.
Refs: 10, 11, 35, 46.

## PR-000.4 — Decision Log & Learning Loop
Modules: `runtime/decision_log.py`, `learning/{loop,candidate_extractor,promotion,feedback}.py`.
Contracts: `DecisionLogEntry, LearningCandidate, LearningOutcome, RuntimeFeedback, ImprovementProposal`.
Responsibilities: record why runtime selected workflow/profile/provider/skill/harness/context; extract learning candidates post-run; feed Memory Harness + Runtime Intelligence; produce improvement proposals; never persist chain-of-thought.
Acceptance: Decision Log per run; learning candidates classified; memory promotion stays governed by Memory Harness; improvement proposals require benchmark evidence; Studio can display decision timeline.
Refs: 09, 11, 18, 32, 44.

# 10. Final 1.0 Mandatory Gate

Not ready until this works reliably:

```bash
opencontext init --profile balanced
opencontext doctor
opencontext index
opencontext run "Fix failing test" --workflow auto
```

Expected: usable config; capability detection; KG built/initialized; OC Flow for localized bugfix; SDD for formal/high-risk; minimal context; safe small mutation; local inspection; bounded diagnosis; escalation when needed; artifacts+receipts persisted; cost/confidence reported; memory/KG updated at consolidation; actionable summary; resumable. Any failure → not 1.0-ready.
