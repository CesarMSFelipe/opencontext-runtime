# SDD Workflow Architecture

Version 1.0 (Draft)

This document defines SDD as a first-class workflow executed by the OpenContext Runtime.

## Purpose
SDD is the formal engineering workflow for medium and high-risk work requiring planning, architecture, traceability and review.

## Canonical Phases

Explore -> Propose -> Specification -> Design -> Tasks -> Apply -> Verify -> Review -> Archive

These phases remain unchanged.

## Principles
- Executed entirely through the Runtime.
- Every phase has contracts, artifacts, receipts and events.
- Personas, Skills and Harnesses are shared with OC Flow.
- Policies and Runtime governance are always enforced.
- Acceptance criteria must exist before Apply.

## Artifacts
- Exploration Report
- Proposal
- Specification
- Design
- Task Graph
- Apply Receipts
- Verification Report
- Review Report
- Archive Summary

## Definition of Done
The workflow completes only after verification, review, consolidation, artifact persistence, receipt generation and KG/Memory updates.
