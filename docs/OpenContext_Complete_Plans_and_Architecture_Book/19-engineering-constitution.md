# OpenContext Engineering Constitution
## Version 1.0 (Draft)
### Document ID
OC-CONSTITUTION-001

# Purpose

This document defines the non-negotiable engineering principles governing OpenContext.

## Principles

1. Runtime before prompts.
2. Contracts before conventions.
3. Evidence before assumptions.
4. Local validation before extra inference.
5. Small context before large context.
6. Quality before speed.
7. Observability before automation.
8. Safety before autonomy.
9. Evolution through benchmarks.
10. Backwards compatibility whenever practical.

## Runtime Rules

- Runtime executes.
- Workflows orchestrate.
- Personas own responsibilities.
- Skills perform reusable work.
- Harnesses validate.
- Policies govern.

## Context Rules

- Context is budgeted.
- KG precedes repository traversal.
- Memory precedes broad file loading.
- Snippets precede whole files.

## Mutation Rules

- Every mutation is attributable.
- Every mutation creates a receipt.
- Rollback should exist whenever feasible.

## Verification Rules

- Local verification precedes additional inference.
- Diagnosis is bounded.
- Verification failures are never ignored.

## Memory Rules

- Evidence-backed only.
- No chain-of-thought persistence.
- Supports supersession.

## KG Rules

- Provenance required.
- Incremental updates preferred.
- Confidence is explicit.

## Release Gates

Stable releases require:

- first-run benchmark
- SDD benchmark
- OC Flow benchmark
- security benchmark
- documentation
- migration notes

## Amendments

Constitution changes require:

- ADR
- benchmark evidence
- migration plan
- approval

## Final Statement

The Constitution defines the permanent engineering rules of OpenContext.
